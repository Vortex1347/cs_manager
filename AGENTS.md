# CS Manager — Godot 4 Project

## Что это

Тактический менеджер в стиле CS:GO. Вид сверху, 3D low-poly арт, 10 ботов (5 CT vs 5 T) играют матч как в реальном CS. Игрок смотрит сверху, паузит, управляет тактикой, улучшает ботов.

## Стек

- **Godot 4.x** (GDScript)
- Никаких внешних зависимостей без необходимости
- Все сцены — текстовые `.tscn` файлы
- Все скрипты — `.gd` файлы

## Правила кода

### Обязательно
- Каждый файл начинается с комментария: что делает, какие зависимости
- Имена переменных — английские snake_case
- Сигналы предпочтительнее прямых вызовов между объектами
- Никогда не хардкодить числа — только через константы или `@export`

### Архитектура ботов (FSM)
Каждый бот живёт в одном из состояний:
```
IDLE → PATROL → SPOTTED_ENEMY → ENGAGE → RETREAT → DEAD
```
Переходы через `bot_brain.gd`. Параметры бота в `bot_stats.gd`.

### Параметры бота (bot_stats.gd)
```gdscript
@export var aim_level: int = 1        # 1-10, влияет на разброс
@export var reaction_time: float = 0.8 # секунды до первого выстрела
@export var game_sense: int = 1       # понимание позиций, тактики
@export var aggression: float = 0.5   # 0=пассивный, 1=агрессивный
```

### Система улучшений
Игрок видит каждого бота, его статы, может потратить очки на:
- `aim_level` — точнее стреляет
- `reaction_time` — быстрее реагирует
- `game_sense` — лучше позиционируется, холдит углы
- Покупка гранат в начале раунда

### Пауза и управление
- `Space` — пауза/анпауза
- В паузе: можно переставить ботов на другие позиции начала раунда
- Можно переключить стратегию команды: rush A / rush B / split / default / eco

## Визуал

**Стиль:** Low-poly, фанки поп, яркие цвета.
- CT: синие тона
- T: оранжево-жёлтые тона
- Карта: нейтральные бежево-серые
- Камера: строго сверху, лёгкий изометрический угол (45° или 60°)
- Тени плоские, освещение мягкое

**Никакого реализма.** Модели — максимально упрощённые капсулы или низкополигональные фигуры.

## Геймплей и механики

### Раунд
1. Фаза покупки (15 сек): боты автоматически покупают по бюджету, игрок может докупить гранаты
2. Раунд (1:55): боты играют по своим статам и выбранной стратегии
3. Конец раунда: подсчёт, деньги, экран статистики
4. Повтор до 16 побед одной команды (30 раундов макс)

### Противники
Разный уровень сложности:
- `EASY`: aim 1-2, reaction 1.2s, game_sense 1
- `MEDIUM`: aim 4-5, reaction 0.7s, game_sense 4
- `HARD`: aim 7-8, reaction 0.35s, game_sense 7
- `PRO`: aim 9-10, reaction 0.2s, game_sense 10

### Гранаты
- `smoke_grenade.gd` — блокирует линии видимости (NavMesh обходит)
- `flash_grenade.gd` — слепит ботов в радиусе на N секунд
- `frag_grenade.gd` — урон в радиусе

## Что делать при каждом новом запросе

1. Сначала проверь существующий код через контекст
2. Не переписывай то, что работает — только дополняй
3. Если добавляешь систему — сначала создай файл скрипта, потом подключи в сцену
4. После каждого изменения выводи: что изменил и где

## Первый шаг при старте проекта

Если проект пустой — начни с этого порядка:
1. `project.godot` с базовыми настройками
2. `game_manager.tscn` + `game_manager.gd`
3. `dust2.tscn` — плоская карта с коллизиями и навигационной сеткой
4. `bot.tscn` + `bot_brain.gd` с базовым FSM (patrol + shoot)
5. Камера сверху
6. HUD с кнопкой паузы

Не делай всё сразу. Каждый шаг должен запускаться и работать.
